Поменяйте модель, подправьте промпт, обновите версию фреймворка, потом запустите `whatbroke` и увидите, что именно изменилось: какие вызовы инструментов исчезли, какие аргументы поплыли, куда сдвинулись стоимость и задержка, какие ответы перевернулись.

Текстовый diff этого не видит. Агент может сказать «ваша подписка отменена» и при этом молча пропустить вызов `cancel_subscription`. Слова выглядят нормально. Поведение сломано.

![вывод whatbroke: пропавший вызов инструмента и изменившаяся сумма возврата после смены модели](../.github/demo.svg)

Это реальный случай после перехода на более дешёвую модель. Агент стал на 75% дешевле, по ощущениям всё было в порядке, но подписки он отменять перестал. А ещё начал возвращать $425 вместо $42.50.

## Установка

```
npm install -g whatbroke
```

Или запускайте напрямую:

```
npx whatbroke diff before.jsonl after.jsonl
```

Попробуйте прямо сейчас на примерах из репозитория:

```
git clone https://github.com/arthi-arumugam-git/whatbroke
cd whatbroke && npm install && npm run build
node dist/cli.js diff examples/support-agent-gpt4o.jsonl examples/support-agent-gpt5mini.jsonl
```

## Как это работает

1. Запишите трейс работы агента (обычный JSONL-файл, одно событие на строку).
2. Поменяйте что-нибудь. Модель, промпт, версию фреймворка, описания инструментов, что угодно.
3. Запишите трейс новой версии на той же задаче.
4. `whatbroke diff old.jsonl new.jsonl`

whatbroke сопоставляет запуски по id, выравнивает вызовы инструментов внутри каждого запуска и сообщает:

| Находка | Серьёзность |
|---|---|
| Запуск начал падать, пропал вызов инструмента, инструмент стал возвращать ошибку, исчез вывод, пропал запуск | breaking |
| Изменились аргументы инструмента, появились новые вызовы, поменялся порядок, изменился вывод, регрессия задержки или стоимости | changed |
| Сменилась модель, сильно скакнули токены, запуск стал успешным | info |

Код выхода равен 1, если нашлось что-то breaking, так что можно сразу ставить в CI:

```yaml
- run: node run-agent-suite.js --out traces/current.jsonl
- run: npx whatbroke diff traces/baseline.jsonl traces/current.jsonl --md >> "$GITHUB_STEP_SUMMARY"
```

`--fail-on warning`, если хотите строже, `--fail-on never`, если нужен только отчёт.

## Нестабильные агенты

Агент никогда не делает одно и то же дважды, поэтому одиночное сравнение до/после может приписать вашему изменению шум, который агент выдавал и так. Запишите каждый сценарий несколько раз и добавьте суффиксы к id:

```
refund-flow#1, refund-flow#2, refund-flow#3
```

whatbroke замечает суффиксы, сравнивает каждый образец «до» с каждым образцом «после» и проставляет каждой находке частоту:

```
! issue_refund called with different args (amount) (6/9 run pairs)
```

Всё, что скачет и между двумя *базовыми* образцами, понижается до flaky info: агент вёл себя так и до вашего изменения. Breaking-находки с частотой меньше половины смягчаются до warning. Что осталось, то и есть сигнал.

## Запись трейсов

Формат трейса намеренно скучный: JSONL, который можно написать на любом языке за десять минут.

```jsonl
{"type":"run_start","run":"refund-flow","meta":{"model":"gpt-4o"}}
{"type":"llm_call","run":"refund-flow","model":"gpt-4o","latency_ms":900,"tokens":{"input":512,"output":128},"cost_usd":0.004}
{"type":"tool_call","run":"refund-flow","name":"lookup_order","args":{"order_id":"A-1042"}}
{"type":"output","run":"refund-flow","content":"Refund issued."}
{"type":"run_end","run":"refund-flow","status":"ok"}
```

Быстрее всего получить трейс через прокси. Ноль изменений в коде, любой язык:

```
whatbroke record --out traces/current.jsonl
```

Дальше направьте агента на прокси и запускайте как обычно:

```
OPENAI_BASE_URL=http://127.0.0.1:4141/v1 node my-agent.js
# или
ANTHROPIC_BASE_URL=http://127.0.0.1:4141 node my-agent.js
```

Каждый вызов LLM, каждый вызов инструмента и финальный ответ попадают в трейс. Стриминг работает, ответы проходят насквозь нетронутыми. Если гоняете несколько сценариев, отправляйте заголовок `x-whatbroke-run` в каждом запросе, чтобы дать запускам имена.

Если вы в Node, SDK оборачивает ваш существующий клиент и записывает всё сам:

```ts
import { Recorder } from "whatbroke";
import OpenAI from "openai";

const rec = new Recorder({ file: "traces/current.jsonl", run: "refund-flow" });
const openai = rec.wrapOpenAI(new OpenAI());

// используйте openai как раньше; вызовы llm и инструментов записываются
await runMyAgent(openai);

rec.output(finalAnswer);
rec.end("ok");
```

`rec.wrapAnthropic(client)` делает то же для Anthropic SDK. На все остальные случаи есть `rec.llmCall()`, `rec.toolCall()`, `rec.output()`, `rec.end()`, либо пишите JSONL сами.

## Опции

```
whatbroke diff <before.jsonl> <after.jsonl>

  --json              машиночитаемый вывод
  --md                вывод в markdown, вставляйте в комментарий к PR
  --fail-on <level>   выход с кодом 1 при: breaking (по умолчанию), warning, never
  --latency <ratio>   помечать регрессии задержки выше этого коэффициента (по умолчанию 1.5)
  --cost <ratio>      помечать рост стоимости выше этого коэффициента (по умолчанию 1.25)
  --no-outputs        не сравнивать финальные ответы

whatbroke record --out <trace.jsonl>

  --port <n>          порт (по умолчанию 4141)
  --run <name>        id запуска, если заголовок x-whatbroke-run не пришёл
  --target <url>      пересылать всё на этот адрес
```

## Почему не просто evals?

Используйте и то и другое. Evals оценивают каждую версию по рубрике. whatbroke отвечает на другой вопрос: что именно изменилось между двумя версиями, на уровне вызовов инструментов, без рубрики и без судьи, которому надо платить. Это то, что запускают через пять минут после выхода новой модели, прежде чем решать, нужно ли вообще гонять eval-набор.

Детерминированный, офлайн, без API-ключей, без аккаунтов. Ваши трейсы никогда не покидают вашу машину.
